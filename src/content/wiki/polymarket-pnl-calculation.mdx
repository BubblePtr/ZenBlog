# Polymarket PnL Calculation（盈亏计算）

Polymarket 生态中最普遍的错误：几乎所有第三方 PnL 追踪工具都算错了。

## 陷阱示例

**操作流程：**
1. 花 $50 执行 Split → 得到 50 YES + 50 NO
2. 花 $20 以 $0.40 买入 50 个 YES
3. 以 $0.35 卖出 50 个 NO → 收回 $17.50
4. 持仓：100 个 YES

**错误算法（大多数工具）：**
- 只看交易记录：买入 $20 / 50 = $0.40/个
- 显示利润：100 × $0.60 - $20 = **$40**

**正确算法：**
- 总现金流出：$50（Split）+ $20（买入）= $70
- 总现金流入：$17.50（卖出 NO）
- 净投入：$70 - $17.50 = $52.50
- 真实成本：$52.50 / 100 = $0.525/个
- 真实利润：100 × $0.60 - $52.50 = **$7.50**

差额 $32.50 是"幻觉利润"。

## 正确公式

> 总盈亏 = Σ(卖出收入) + Σ(Merge 收入) + Σ(Redeem 收入) - Σ(买入支出) - Σ(Split 支出) + 当前持仓市值

## 常见错误来源

1. **忽略 <WikiLink slug="split-operation" label="Split" /> 成本**：Split 不是交易，很多工具不记录
2. **忽略卖出互补代币的收益**：卖掉 NO 获得的 pUSD 未被计入
3. **Redeem 后持仓被抹除**：赢的仓位 Redeem 后从历史记录中消失，只留下亏损的仓位

排行榜上某些大佬显示亏损几百万，实际上可能赚得盆满钵满。

[@insidersdotbot](https://insiders.bot) 声称已解决此问题，使用正确的 PnL 算法。
