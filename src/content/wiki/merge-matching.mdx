# Merge Matching（合并匹配）

<WikiLink slug="mint-matching" /> 的反向过程：两个卖家可以成交，销毁代币换回资金。

## 机制

场景：市场即将结束，大家都在平仓。Alice 以 $0.60 卖 100 个 YES，Bob 以 $0.40 卖 100 个 NO。

1. 从 Alice 拿走 100 个 YES
2. 从 Bob 拿走 100 个 NO
3. 调用 `_merge` 将 YES+NO 对销毁，从金库解锁 $100 pUSD
4. $60 pUSD 给 Alice，$40 pUSD 给 Bob

## 特征

- **通缩机制**：系统总代币供应量减少
- **退出通道**：即使没有"接盘侠"，只要 YES 和 NO 卖家价格能凑够 $1.00，依然可以套现离场
- **价格约束**：卖家出价之和 ≤ $1.00（让出的空间即赎回价值）

## 市场生命周期中的角色

市场晚期（结果逐渐明朗），merge 主导。多空双方通过此机制销毁代币换回资金，总供应量下降。
